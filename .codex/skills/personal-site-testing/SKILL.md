# Personal Site Testing

## Quick Start

Use this workflow from `/home/ubuntu/personal-site`.

1. Install and sync dependencies.

    uv sync --all-extras

2. Run the Python test suite.

    uv run pytest -q

3. Generate visual screenshots for the supported viewports and color schemes.

    scripts/ui_screenshots.sh

4. Review output under `artifacts/playwright/*.png` for:
- alignment shifts
- overflow/horizontal scroll
- mobile stacking issues
- low-contrast states

## Functional Checks

Run these checks after any route, content, or metadata edits.

1. Core routes return 200:
- `/`
- `/blog`
- `/coursework`
- `/feed.xml`
- `/robots.txt`
- `/sitemap.xml`
- `/llms.txt`

2. Theme toggle persists a cookie and updates `data-theme` in HTML.

3. Blog post pages render canonical metadata and article schema.

## SEO and Crawler Checks

Use quick smoke checks when metadata or crawler policy changes.

    curl -fsS http://127.0.0.1:8000/robots.txt
    curl -fsS http://127.0.0.1:8000/sitemap.xml
    curl -fsS http://127.0.0.1:8000/llms.txt
    curl -fsS http://127.0.0.1:8000/feed.xml

Verify:
- sitemap URL points to canonical host
- key post URLs exist in sitemap and feed
- OpenAI + major AI crawler directives are present in robots

## Responsive and Accessibility Guardrails

For each changed page in screenshots and manual run:

- desktop (`1280x800`), tablet (`768x1024`), mobile (`390x844`)
- light and dark color schemes
- focus-visible states work with keyboard navigation
- tap targets remain usable on mobile (44px class target)
- no layout overflow, clipped text, or overlapping controls

## Static Cache-Busting Note

If CSS or JS under `app/static/` is changed, restart the app process before re-testing to refresh static digests generated by `static_url()`.

## Completion Checklist

- `uv run pytest -q` passes
- screenshot script completes without errors
- crawler and feed endpoints return expected content
- no responsive regressions in screenshot outputs
